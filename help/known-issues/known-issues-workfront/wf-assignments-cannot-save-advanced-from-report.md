# Affectations : impossible d’enregistrer la modification d’affectation avancée à partir d’un rapport

Lorsque l’utilisateur tente de modifier une affectation avancée à partir d’un rapport, la modification n’est pas enregistrée et le message suivant apparaît :

« 422 : APIModel INTERNAL ne prend pas en charge le champ removeOldAssignments »

**Solution de contournement**

Vous pouvez modifier les affectations avancées à partir de la liste des tâches du projet.

_Premier signalement le 4 février 2026._
